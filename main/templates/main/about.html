О проекте

Что это за проект?

Этот проект — мой пет-проект, представляющий собой веб-приложение для отслеживания и анализа криптовалютных токенов на различных блокчейнах. Основная цель проекта — предоставить пользователям удобный и функциональный инструмент для мониторинга токенов в реальном времени, а также управления избранными токенами в личном кабинете.

Приложение включает такие разделы, как просмотр только что появившихся токенов, подробная информация о токене, возможность сохранения избранных токенов и управления ими, а также административную панель для редактирования данных о токенах и управления их статусом.

Используемые технологии

В разработке этого проекта я использовал различные современные технологии и инструменты. Это помогло мне улучшить навыки в области веб-разработки и научиться работать с различными стеками технологий. Ниже перечислены основные технологии, которые я использовал:

Django

Основной фреймворк для разработки веб-приложения. С его помощью реализованы такие функции, как создание страниц сайта, управление личными кабинетами пользователей, а также разработка административной панели для редактирования информации о токенах.

Личный кабинет

Пользователи могут регистрироваться через Google, GitHub или Facebook, создавать и просматривать свои сохраненные подборки токенов, что реализовано с использованием Django и интеграции с внешними API для авторизации.

Redis

Redis используется для организации кеширования данных, что позволяет хранить список токенов с наивысшей рыночной капитализацией за 24 часа, избегая лишних запросов к базе данных и ускоряя работу приложения. Кроме того, Redis задействован для отображения страницы последних токенов.

Celery

Celery используется для асинхронной обработки задач, таких как обновление кеша Redis при добавлении новых токенов и сбор дополнительной информации о токенах после их появления. В связке с Celery активно применяются Lua-скрипты для оптимизации процессов записи и чтения из Redis.

PostgreSQL

В качестве основной базы данных используется PostgreSQL, где хранятся данные о токенах, пользователях, их избранных токенах и категориях. Структура базы данных аналогична структуре, используемой в Dexscreener, что позволяет эффективно управлять данными и выполнять сложные запросы.

Asyncio и Threads

Asyncio используется для асинхронных запросов к внешним API, таким как Dexscreener, и для обработки данных в режиме реального времени через вебсокеты. Потоки (Threads) задействованы для параллельного запуска слушателей блокчейнов и решения задач, связанных с синхронизацией данных через вебсокеты.

Websockets и Django Channels

Реализация функционала в режиме реального времени, например, появление новых токенов на главной странице, была осуществлена с использованием Websockets и Django Channels. Эти технологии позволяют обновлять интерфейс пользователей без необходимости перезагрузки страницы.

Docker

Весь проект разворачивается с использованием Docker, что позволяет легко управлять инфраструктурой и развертыванием проекта. Проект разделен на несколько контейнеров: основной сервис (сайт), база данных, сервер проксирования запросов к Dexscreener и сервер обработки задач Celery.
